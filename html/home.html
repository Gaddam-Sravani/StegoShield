How StegoShield Works

Step 1: Cryptographic Fortification

Before any data is hidden, it undergoes a rigorous encryption process. StegoShield utilizes AES-GCM 256-bit, an authenticated encryption standard that provides both confidentiality and data integrity. To protect against brute-force attacks, your password is processed through the PBKDF2 algorithm with 100,000 iterations and a unique salt, transforming it into a high-entropy master key.

Step 2: Advanced Media Embedding

🖼️ Image Steganography

Utilizing Least Significant Bit (LSB) Substitution, we hide data within the color channels of a PNG image. Each pixel consists of Red, Green, and Blue (RGB) values. By modifying only the lowest bit of these 8-bit values, we can store binary data without altering the visual appearance of the pixel. The mathematical change is so minute (1 part in 256) that the human eye cannot detect the alteration even under high magnification.

🎵 Audio Steganography

For 16-bit WAV files, we implement PCM Amplitude Modulation. We convert sound waves into a series of numerical samples. By hiding the secret bits within the LSB of these samples, we are essentially hiding your data in the "whispers" of the audio stream. Since the modification affects only 1 out of 65,536 possible amplitude levels per sample, the resulting noise is physically below the threshold of human hearing.

🎬 Video Steganography

Video files are highly sensitive to pixel manipulation due to aggressive lossy compression (VP8/H.264). To combat this, StegoShield uses Structural Binary Appending. Instead of hiding bits in the frames where they would be "smudged" out, we inject the encrypted payload immediately following the video's End-of-File (EOF) marker. Standard media players ignore any data following this marker, but our extractor scans the binary architecture to retrieve it with 100% bit-accuracy.

📄 Documents(.pdf, .html, .docx)

This module treats complex document structures as generic Binary Stream Containers. Document readers like Adobe Acrobat or Microsoft Word look for specific structural tags to start and stop rendering. We utilize EOF Cloaking to stitch your encrypted secret, a unique Stego Marker, and the original filename metadata to the end of the document's binary stream. This allows the document to remain fully functional and editable while secretly carrying a separate, hidden file system within its architecture.

Step 3: Secure Reconstruction

During extraction, the system scans the carrier’s binary map for a high-entropy Stego Marker. Once found, it isolates the encrypted blob, parses the metadata header to recover the original filename/extension, and initiates the AES-GCM decryption sequence. If the password is correct and the data hasn't been tampered with, the original file is reconstructed in RAM and offered for download.

Core Concepts

Cryptography ensures your message cannot be read. Steganography ensures your message cannot be found.

By combining both, StegoShield provides a two-layer defense that protects not only the content of your communication but the metadata of the transmission itself.
